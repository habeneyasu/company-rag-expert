# Knowledge Base Data

This directory contains the company knowledge base organized by category for use with the RAG (Retrieval-Augmented Generation) system.

## Directory Structure

```
knowledge-base/
├── company/          # Company-wide policies, mission, and organizational information
├── contracts/        # Legal templates and contract documents
├── employees/        # Employee-facing documents and procedures
└── products/        # Product documentation, APIs, and technical specifications
```

## Content Overview

### Company Documents
- **company_policy.md**: Comprehensive company policy handbook covering code of conduct, workplace policies, benefits, and more
- **mission_and_values.md**: Company mission, vision, core values, and strategic priorities
- **organizational_structure.md**: Organizational chart, department structure, and reporting relationships

### Contracts
- **nda_template.md**: Non-disclosure agreement template
- **service_agreement_template.md**: Service agreement template for client engagements
- **vendor_agreement_template.md**: Vendor services agreement template

### Employees
- **onboarding_guide.md**: Complete employee onboarding process and checklist
- **performance_review_process.md**: Performance review procedures, ratings, and development planning
- **expense_reimbursement_policy.md**: Expense reimbursement guidelines and procedures

### Products
- **product_catalog.md**: Complete product catalog with features, pricing, and specifications
- **api_documentation.md**: Comprehensive API documentation with endpoints and examples
- **faq.md**: Frequently asked questions about products and services
- **technical_specifications.md**: Technical requirements, architecture, and system specifications

## Usage

This knowledge base is designed to:
- Provide comprehensive context for RAG system testing
- Cover diverse document types and structures
- Include realistic business content across multiple domains
- Support various query types and retrieval scenarios

## Document Metadata

Each document includes:
- Document owner
- Last updated date
- Version number
- Classification (where applicable)

## Best Practices

When adding new documents:
1. Place documents in the appropriate category directory
2. Use descriptive, consistent naming conventions
3. Include metadata at the end of each document
4. Maintain consistent formatting and structure
5. Update this README when adding new categories
